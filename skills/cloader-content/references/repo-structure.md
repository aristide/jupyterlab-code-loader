## Directory layout

```
examples-registry/
├── registry.json                        # Auto-generated master index
├── domains/
│   ├── {domain-id}/
│   │   ├── manifest.yaml               # Domain metadata + content list
│   │   ├── requirements.txt            # Python deps (optional)
│   │   ├── install.R                   # R deps (optional)
│   │   ├── code/
│   │   │   ├── en/                     # English (always complete)
│   │   │   │   ├── example.ipynb
│   │   │   │   ├── script.py
│   │   │   │   └── analysis.R
│   │   │   ├── fr/                     # French (partial OK)
│   │   │   │   └── example.ipynb
│   │   │   └── {locale}/
│   │   │       └── ...
│   │   └── snippets/
│   │       ├── en/
│   │       │   └── helpers.json
│   │       └── fr/
│   │           └── helpers.json
│   └── {another-domain}/
│       └── ...
├── snippets/                            # Cross-domain shared snippets
│   ├── en/
│   │   └── common.json
│   └── fr/
│       └── common.json
├── i18n/                                # UI label translations
│   ├── en.json
│   └── fr.json
└── meta/                                # Optional metadata
    └── CONTRIBUTING.md
```

## registry.json

Top-level index file, auto-generated by `build_registry.py`.

```json
{
  "version": "1.0.0",
  "generated_at": "2026-03-24T12:00:00Z",
  "repo_url": "https://github.com/org/examples-registry",
  "domains": [
    {
      "id": "geospatial",
      "name": "Geospatial Analysis",
      "description": "GIS workflows with GeoPandas and Rasterio",
      "tags": ["gis", "maps", "raster"],
      "locales": ["en", "fr"],
      "notebook_count": 3,
      "script_count": 2,
      "snippet_count": 1,
      "last_updated": "2026-03-24T12:00:00Z"
    }
  ]
}
```

### Fields

| Field          | Type   | Description                     |
| -------------- | ------ | ------------------------------- |
| `version`      | string | Semver version of the registry  |
| `generated_at` | string | ISO 8601 timestamp              |
| `repo_url`     | string | Git repository URL              |
| `domains`      | array  | Array of domain summary objects |

### Domain summary fields

| Field            | Type     | Description                            |
| ---------------- | -------- | -------------------------------------- |
| `id`             | string   | URL-safe slug (matches directory name) |
| `name`           | string   | Human-readable display name            |
| `description`    | string   | Short description                      |
| `tags`           | string[] | Domain-level topic tags                |
| `locales`        | string[] | Available locale codes                 |
| `notebook_count` | integer  | Number of notebook files in manifest   |
| `script_count`   | integer  | Number of script files in manifest     |
| `snippet_count`  | integer  | Number of snippet files in manifest    |
| `last_updated`   | string   | ISO 8601 timestamp                     |

## manifest.yaml

Each domain has a `manifest.yaml` in its root directory.

```yaml
id: geospatial
name: Geospatial Analysis
version: 1.0.0
description: >
  GIS workflows, raster analysis, spatial joins,
  and geocoding with GeoPandas, Rasterio, and R/sf.
locales: [en, fr]
authors:
  - name: Alice Martin
    github: alicemartin
tags: [gis, maps, raster, vector]

code:
  - file: 01_intro_geopandas.ipynb
    title: Introduction to GeoPandas
    description: Load shapefiles and perform spatial joins
    type: notebook
    code_lang: python
    difficulty: beginner
    estimated_time: 15min
    tags: [geopandas, shapefile, intro]
    kernel: python3
    i18n:
      fr:
        title: "Introduction \u00e0 GeoPandas"
        description: 'Charger des shapefiles et effectuer des jointures spatiales'

  - file: batch_geocode.py
    title: Batch Geocoding Script
    description: Geocode addresses from a CSV file
    type: script
    code_lang: python
    difficulty: intermediate
    tags: [geocoding, csv, batch]

snippet_files:
  - file: coordinate_transforms.json
    title: Coordinate Transformations
    tags: [crs, projection]
    i18n:
      fr:
        title: "Transformations de coordonn\u00e9es"
```

### Code item fields

| Field            | Type     | Required | Values                                      |
| ---------------- | -------- | -------- | ------------------------------------------- |
| `file`           | string   | Yes      | Filename relative to `code/{locale}/`       |
| `title`          | string   | Yes      | Display title (English default)             |
| `description`    | string   | Yes      | Short description (English default)         |
| `type`           | enum     | Yes      | `notebook`, `script`                        |
| `code_lang`      | enum     | Yes      | `python`, `r`, `julia`, `bash`              |
| `difficulty`     | enum     | No       | `beginner`, `intermediate`, `advanced`      |
| `estimated_time` | string   | No       | e.g., `15min`, `1h`                         |
| `tags`           | string[] | No       | Topic tags                                  |
| `kernel`         | string   | No       | Jupyter kernel name (for notebooks)         |
| `i18n`           | object   | No       | Per-locale `{title, description}` overrides |

### Snippet file reference fields

| Field   | Type     | Required | Description                                    |
| ------- | -------- | -------- | ---------------------------------------------- |
| `file`  | string   | Yes      | JSON filename relative to `snippets/{locale}/` |
| `title` | string   | Yes      | Display title                                  |
| `tags`  | string[] | No       | Topic tags                                     |
| `i18n`  | object   | No       | Per-locale `{title}` overrides                 |

## Snippet JSON files

Located at `domains/{domain_id}/snippets/{locale}/{filename}.json` or
`snippets/{locale}/{filename}.json` for shared snippets.

```json
{
  "id": "coordinate_transforms",
  "title": "Coordinate Transformations",
  "lang": "en",
  "snippets": [
    {
      "id": "reproject_gdf",
      "title": "Reproject a GeoDataFrame",
      "description": "Convert a GeoDataFrame from one CRS to another",
      "code_lang": "python",
      "lang": "en",
      "tags": ["crs", "reproject"],
      "code": "gdf_utm = gdf.to_crs(epsg=32633)\nprint(f'CRS: {gdf_utm.crs}')",
      "imports": ["import geopandas as gpd"]
    }
  ]
}
```

### File-level fields

| Field      | Type   | Required | Description                        |
| ---------- | ------ | -------- | ---------------------------------- |
| `id`       | string | Yes      | Unique file identifier             |
| `title`    | string | Yes      | Display title                      |
| `lang`     | string | Yes      | Content language code (e.g., `en`) |
| `snippets` | array  | Yes      | Array of snippet objects           |

### Snippet object fields

| Field         | Type     | Required | Description                                |
| ------------- | -------- | -------- | ------------------------------------------ |
| `id`          | string   | Yes      | Unique within file                         |
| `title`       | string   | Yes      | Display title                              |
| `description` | string   | No       | Short description                          |
| `code_lang`   | enum     | Yes      | `python`, `r`, `julia`, `bash`             |
| `lang`        | string   | Yes      | Content language code                      |
| `tags`        | string[] | No       | Topic tags (default: `[]`)                 |
| `code`        | string   | Yes      | The snippet code                           |
| `imports`     | string[] | No       | Required import statements (default: `[]`) |

## i18n UI labels

Located at `i18n/{locale}.json`. Contains translated UI chrome strings.

```json
{
  "sidebar.title": "Examples & snippets",
  "tab.code": "Code examples",
  "tab.snippets": "Snippets",
  "search.code.placeholder": "Search code examples...",
  "search.snippets.placeholder": "Search snippets...",
  "status.lastSync": "Last synced {time} ago",
  "status.refresh": "Refresh",
  "action.copyToWorkspace": "Copy to workspace",
  "action.openInEditor": "Open in editor",
  "action.insertSnippet": "Insert snippet",
  "action.copyToClipboard": "Copy to clipboard",
  "difficulty.beginner": "beginner",
  "difficulty.intermediate": "intermediate",
  "difficulty.advanced": "advanced",
  "locale.fallbackNotice": "Translation unavailable \u2014 English version",
  "locale.translated": "Translated",
  "kernel.detected": "Kernel {name} detected",
  "kernel.none": "No notebook active",
  "filter.shown": "{n} shown",
  "filter.hidden": "{n} hidden ({lang})",
  "filter.noFilter": "no filter"
}
```

All keys from `en.json` should be present in every locale file. The `{time}`, `{name}`,
`{n}`, `{lang}` placeholders are replaced at runtime.

## Locale resolution

The extension resolves content files using a **three-step fallback chain**:

1. `{base}/{requested_locale}/{filename}` — if it exists, serve it (`_is_translated = true`)
2. `{base}/{default_locale}/{filename}` — configured default locale
3. `{base}/en/{filename}` — ultimate fallback to English

This applies independently to each file. A French user may see 3 of 5 notebooks in French
and 2 in English.

**Rules:**

- `en/` must contain every file listed in the manifest
- Other locale directories can be partial
- Filenames must match exactly between locales (same name = same content, different language)

## Supported file types

| Extension | Type     | code_lang   | Opens with      |
| --------- | -------- | ----------- | --------------- |
| `.ipynb`  | Notebook | From kernel | Notebook editor |
| `.py`     | Script   | `python`    | Text editor     |
| `.R`      | Script   | `r`         | Text editor     |
| `.Rmd`    | Script   | `r`         | Text editor     |
| `.jl`     | Script   | `julia`     | Text editor     |
| `.sh`     | Script   | `bash`      | Text editor     |

## Naming conventions

| Entity         | Format                                 | Examples                               |
| -------------- | -------------------------------------- | -------------------------------------- |
| Domain ID      | lowercase + hyphens                    | `geospatial`, `data-science`, `ml-ops` |
| Snippet ID     | lowercase + underscores                | `reproject_gdf`, `read_csv_chunked`    |
| Locale code    | BCP 47                                 | `en`, `fr`, `es`, `pt-BR`              |
| Code filenames | descriptive, consistent across locales | `01_intro.ipynb`, `batch_geocode.py`   |

## Kernel mapping

The extension maps Jupyter kernel names to `code_lang` for filtering:

| Kernel pattern                                                        | code_lang            |
| --------------------------------------------------------------------- | -------------------- |
| `python3`, `python`, `conda-*python*`, `ipykernel`, `xpython`, `pypy` | `python`             |
| `ir`, `r`, `r4`                                                       | `r`                  |
| `julia-*`, `julia`                                                    | `julia`              |
| `bash`, `sh`                                                          | `bash`               |
| Unknown / null                                                        | No filter (show all) |
